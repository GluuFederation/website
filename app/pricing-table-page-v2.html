FAQ's

What are the basic server requirements for a Gluu Server deployment?We recommend a virtual machine with at least 2 CPU units, 4 GB of RAM and around 30GB of disk space.

Where can I deploy the Gluu Server?The Gluu Server can be deployed on any cloud provider, including Microsoft Azure, Amazon Web Services (AWS), Google Cloud Engine (GCE), and Digital Ocean. For cloud specific notes, please view our VM prep guide. 

How many Gluu Servers do organizations typically deploy?Organizations most commonly deploy one (production or development), two (production and development), or four or more (at least two production and two development) Gluu servers depending on their requirements for high availability, staging, QA, and fail over.

Does Gluu provide any professional services?No. If your organization needs customization or integration services, we encourage you to reach out to one of our certified partners.

Does Gluu publish its customer list?No. However, the Gluu Server is being used in production at educational institutions, government and non-government organizations, and enterprises all over the world.

Does Gluu support previous releases in production and pre-production environments?Yes, Gluu supports one version prior to the most current Gluu Server release. 

Does Gluu provide any per incident support options?No, Gluu only provides annual support options. For per incident support, please contact one of our preferred partners. 

Is the Gluu Server free to use in production?Yes. The Gluu Server is, and always will be, free to use in production with no per user limits. Check our docs for more information about the license for each of the open source products in use.

Can I cancel my support subscription at any time?Either party may terminate the support contract with 30 days written notice. Any unused balance will be refunded.

What happens if I cancel my support subscription?If you cancel your support subscription you will lose enhanced access to our support services and any unused balance will be refunded.

How many users can Gluu handle, and what is the max speed of concurrent logins?The Gluu Server can be scaled horizontally to handle millions of users. If you need more performance from the database layer, there is a commercial LDAP server called UnboundID Data Store which enables you to split your data into multiple pieces, and effectively route the LDAP traffic using an LDAP proxy. In terms of performance, the rule of thumb we use is 10-20 authentications per core per second.
