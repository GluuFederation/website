Client Software to Secure Applications

  • Learn more
  • Learn more

In addition to oxd, the following 3rd party client software projects can be used to
secure the sign-in flow for web and mobile applications against the Gluu Server.

  • mod_auth_openidc

    Authentication/authorization module for the Apache 2.x HTTP server that authenticates users against an OpenID Connect Provider..

    Learn more

  • Lua Resty OpenIDC

    Library for NGINX implementing the OpenID Connect Relying Party (RP) and the OAuth 2.0 Resource Server (RS) functionality.

    Learn more

  • AppAuth

    Android, iOS and macOS client SDK’s for communicating with OAuth 2.0 and OpenID Connect providers.

    Learn more

  • Javascript Implicit
    Flow Client

    Library to provide OpenID Connect (OIDC) and OAuth2 protocol support for client-side, browser-based JavaScript client applications.

    Learn more

Free and Commercial Strong Authentication

Super Gluu

Free and secure two-factor authentication mobile
application.

Enhance security with two-factor authentication to any
applications that relies on Gluu for login.

  • Learn more
  • Learn more

In addition to Super Gluu, the following 3rd party authentication mechanisms can be
used to fortify access to web and mobile applications that use the Gluu Server for login.

  • Yubikey

    A hardware authentication device that supports one-time passwords, public key encryption and authentication, and the Universal 2nd Factor (U2F) protocol developed by FIDO.

    Learn more

  • DUO Security

    Duo Security provides two-factor authentication as a service to protect against account takeover and data theft.

    Learn more

  • Google Authenticator

    A free service that implements two-step authentication using Time-based One-time Passwords (TOTP).

    Learn more

  • Hyper FIDO

    The HyperFido hardware key supports the FIDO U2F protocol and enables seamless authentication to websites and browser based applications.

    Learn more

  • Lua Resty OpenIDC

    Library for NGINX implementing the OpenID Connect Relying Party (RP) and the OAuth 2.0 Resource Server (RS) functionality.

    Learn more

Don’t see your preferred authentication mechanism here? No problem! Gluu can be configured to use any
authentication mechanism that exposes a web API or supports the FIDO authentication standard.
